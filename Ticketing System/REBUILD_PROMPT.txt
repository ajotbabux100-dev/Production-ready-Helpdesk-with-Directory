================================================================================
  1. WHAT THIS SYSTEM IS
================================================================================

A full-stack internal Helpdesk & IT Ticketing System built for GSH/ISH OMAN.
Staff submit support tickets, agents resolve them, managers oversee departments,
and admins control everything. The system is self-hosted (Windows Server / VPS).

Key principles:
- Role-based access: end_user / agent / manager / admin
- Every ticket belongs to a department and has an SLA policy
- Full audit trail of every action
- In-app notifications + optional email (SMTP) notifications
- Completely white-label: company name, logo, colors, ticket prefix all configurable
- Soft-delete for users and departments (preserve ticket history)

================================================================================
  6. BUSINESS LOGIC
================================================================================

TICKET NUMBERING
  Format: {PREFIX}{SEP}{YEAR}{SEP}{SEQUENCE}  e.g. TKT-2026-00001
  - Fully configurable prefix, separator, year format (YYYY/YY), sequence digits
  - Optional yearly reset of counter
  - Atomic generation with select_for_update to prevent duplicates

SLA ENFORCEMENT
  On ticket create: look up SLAPolicy for (department, priority). Fall back to global
  (department=null, same priority). Set ticket.sla_response_due and sla_resolution_due.
  Ticket model properties is_sla_response_breached and is_sla_resolution_breached compute live.

AUTO-ASSIGNMENT
  On ticket create: if department.routing_mode == 'pool' → no auto-assign (pool visible to all dept members).
  If routing_mode == 'manager' and department.auto_assign_to is set → assign to that person.

ESCALATION
  - Only manager or admin can escalate a ticket.
  - Once escalated, agent cannot change status, reassign, or add assignment.
  - Only manager/admin can act on escalated ticket.
  - Cannot escalate an already-escalated ticket.
  - Triggers ticket_escalated notification to dept manager and admin.

DEPARTMENT POOL ROUTING
  - routing_mode = 'pool': all members of the department see unassigned tickets.
  - Any agent in the dept can claim (assign to themselves) and resolve.
  - routing_mode = 'manager': manager assigns each ticket to a specific agent.

@MENTION / CONTRIBUTORS
  - In ticket comments, type @Name to invite a user.
  - Creates TicketParticipant record for that user.
  - mention_scope on Department controls who can be mentioned:
      'department' = only dept members    'all' = anyone in the system
  - Participants see the ticket even if not the requester or assigned agent.
  - Participant status: active → contributed (after they comment) → exited.

SOFT DELETE
  Users: is_deleted=True, deleted_alias='Deleted User #N', is_active=False.
  - Email freed for reuse by prefixing with 'deleted_' + timestamp.
  - All their tickets and audit logs preserved, display shows deleted_alias.
  Departments: is_deleted=True, deleted_alias='Deleted Dept #N', is_active=False.
  - All dept tickets preserved.

ROLE PERMISSIONS MATRIX
  end_user:  submit own tickets, add comments, view own tickets only
  agent:     all above + view/work assigned tickets + all dept tickets (pool mode)
  manager:   all above + view all dept tickets + assign + escalate + dept management
  admin:     everything + user management + settings + audit log + reports

TICKET VISIBILITY
  Admin:   all tickets
  Manager: all tickets in own department + invited as participant
  Agent:   assigned tickets + all dept tickets (pool) + invited as participant
  User:    own submitted tickets + invited as participant

TICKET STATUS FLOW
  new → assigned → in_progress → pending_user / pending_vendor → resolved → closed
  Any open status → escalated (manager/admin)
  resolved/closed → reopened

FIRST RESPONSE TRACKING
  first_response_at set when agent/manager first adds a non-internal comment.
  Used for SLA response breach calculation.

EMAIL NOTIFICATIONS
  Triggered by: ticket_created, ticket_assigned, status_updated, comment_added,
  ticket_resolved, sla_breach, ticket_escalated.
  Each event type can be toggled on/off in SystemSettings.
  SMTP configured in SystemSettings (host, port, TLS/SSL, credentials).
  Email content includes ticket number, title, status, link.
  notify_on_* flags control which events actually send emails.

IN-APP NOTIFICATIONS
  Created alongside email for every event.
  Bell icon in Topbar shows unread count (polled every 60 seconds).
  Notifications page lists all, with mark-read.

AUDIT LOG
  Every significant action logged: ticket CRUD, status changes, assignments,
  comments, user CRUD, dept CRUD, login/logout.
  Stores: user, action type, ticket, description, old_value, new_value, IP, timestamp.
  Visible to admin only.

  8. ZUSTAND STORE (frontend/app/lib/store.ts)
================================================================================

State:
  user              User | null
  accessToken       string | null
  refreshToken      string | null
  _hasHydrated      boolean   (becomes true after persist rehydrates from localStorage)
  setHasHydrated    (v: boolean) => void
  setAuth           (user, access, refresh) => void   saves to localStorage + state
  clearAuth         () => void                         removes from localStorage + state
  setUser           (user) => void

Persist config:
  name: 'auth-storage'
  partialize: saves user, accessToken, refreshToken (NOT _hasHydrated)
  onRehydrateStorage: () => (state) => state?.setHasHydrated(true)

Dashboard layout guards using _hasHydrated:
  - If !hasHydrated → show spinner (don't render children)
  - If hasHydrated && !user → router.replace('/login')
  - If hasHydrated && user → render full layout

================================================================================
  10. KEY IMPLEMENTATION DETAILS
================================================================================

BACKEND

  Department auto-sets manager:
    users/signals.py: post_save on User — if role changed to 'manager' and user has dept,
    set dept.manager = user automatically. If changed away from manager, clear it.

  SLA application (tickets/sla.py):
    Look up SLAPolicy for (ticket.department, ticket.priority).
    Fall back to (None, ticket.priority) for global default.
    Set ticket.sla_response_due = now + response_time_minutes
    Set ticket.sla_resolution_due = now + resolution_time_minutes

  Auto-assign (tickets/auto_assign.py):
    If dept.routing_mode == 'pool': do nothing (leave assigned_to=None)
    If dept.auto_assign_to is set: ticket.assigned_to = dept.auto_assign_to

  Notification sending (notifications/tasks.py):
    Called after every ticket event. Reads SystemSettings to check:
      - email_enabled flag
      - relevant notify_on_* flag
    Creates Notification record for each recipient.
    If email enabled and flag on, sends email via SMTP.
    Recipients depend on event type:
      ticket_created:  dept manager + admin
      ticket_assigned: assigned agent
      status_updated:  requester + assigned agent
      comment_added:   requester + assigned agent + all participants
      ticket_resolved: requester
      sla_breach:      dept manager + admin
      ticket_escalated: dept manager + admin

  Permissions (users/permissions.py):
    IsAgentOrAbove: user.role in (agent, manager, admin)
    IsManagerOrAbove: user.role in (manager, admin)
    IsAdminUser: user.role == admin

  Media files:
    avatars/ → user profile pictures
    branding/ → company logo, favicon
    attachments/ticket_{id}/ → ticket file attachments
    attachments/comment_{id}/ → comment file attachments

FRONTEND

  Sidebar navigation — role-aware visibility:
    end_user: Dashboard, My Tickets, New Ticket, Notifications
    agent:    + Departments (read-only), Reports (read-only)
    manager:  + Users (read-only), Departments (edit SLA), Reports
    admin:    + Users (full), Settings, Audit Log

  Topbar:
    Shows current page title, unread notification bell (polls /api/notifications/unread_count/ every 60s),
    user avatar (initials), dropdown with Profile + Sign Out

  @mention autocomplete:
    In comment textarea, detect '@' character, show dropdown of matching users.
    Users list filtered by department.mention_scope (dept members only OR all users).
    On select: POST /api/tickets/{id}/invite/ to create TicketParticipant.

  SLA display:
    In ticket detail and list, show colored badges:
      Green if within SLA   Orange if < 1hr remaining   Red if breached

================================================================================
  13. KNOWN ISSUES & WORKAROUNDS
================================================================================

1. Windows path with '&' character (GSH&ISH OMAN):
   npm run dev fails with EPERM on this path.
   WORKAROUND: use direct node invocation:
   node node_modules\next\dist\bin\next dev --port 3000

2. Next.js 15.3.4 + React 19 "invariant expected layout router to be mounted" (E56):
   Root cause: Zustand's onRehydrateStorage fires set() synchronously during
   localStorage hydration, which interrupts Next.js's hydrateRoot() startTransition.
   The dashboard layout hides {children} while _hasHydrated=false, which prevents
   OuterLayoutRouter from running during the dangerous hydration window.
   If this error appears again after future Next.js upgrades, the fix is to ensure
   {children} is not rendered in ANY layout during the initial hydration render.

3. Manager seeing ISE on Departments page:
   DepartmentMinimalSerializer (for non-admin) omits sla_policies field.
   Always use optional chaining: dept.sla_policies?.length instead of dept.sla_policies.length
   All admin-only actions (Edit, Delete, Add) must be hidden behind isAdmin check.
